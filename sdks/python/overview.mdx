# Unsiloed Python SDK

The official Python SDK for [Unsiloed](https://unsiloed.ai) provides a powerful and intuitive interface to interact with the Unsiloed Vision API. Process documents with advanced AI-powered capabilities for parsing, data extraction, classification, and splitting.

## Features

## Quick Start

### Synchronous Usage

Perfect for scripts, notebooks, and traditional Python applications:

```python
from unsiloed_sdk import UnsiloedClient

# Initialize client with API key
with UnsiloedClient(api_key="your-api-key-here") as client:
    # Parse a document
    result = client.parse_and_wait(file="document.pdf")
    print(f"Total chunks: {result.total_chunks}")

    # Extract structured data
    schema = {
        "type": "object",
        "properties": {
            "invoice_number": {"type": "string"},
            "total": {"type": "number"}
        },
        "required": ["invoice_number", "total"],
        "additionalProperties": False
    }
    extract_result = client.extract_and_wait(file="invoice.pdf", schema=schema)
    print(f"Invoice #: {extract_result.result['invoice_number']['value']}")
```

### Async Usage

Perfect for FastAPI apps and concurrent processing:

```python
import asyncio
from unsiloed_sdk import AsyncUnsiloedClient

async def main():
    async with AsyncUnsiloedClient(api_key="your-api-key-here") as client:
        # Process multiple documents concurrently
        results = await asyncio.gather(
            client.parse_and_wait(file="doc1.pdf"),
            client.parse_and_wait(file="doc2.pdf"),
            client.parse_and_wait(file="doc3.pdf"),
        )

        for i, result in enumerate(results, 1):
            print(f"Document {i}: {result.total_chunks} chunks")

asyncio.run(main())
```

## Which Client Should I Use?

| Use Case | Client | Best For |
|----------|--------|----------|
| Scripts, notebooks | `UnsiloedClient` | Quick scripts, data analysis, batch processing |
| Flask, Django | `UnsiloedClient` | Traditional web frameworks with sync views |
| FastAPI, async apps | `AsyncUnsiloedClient` | Modern async frameworks, high concurrency |
| Concurrent processing | `AsyncUnsiloedClient` | Processing multiple documents simultaneously |

## Core Concepts

### Job-Based Processing

All document processing operations in Unsiloed are asynchronous and job-based:

1. **Submit**: Upload a document and get a job ID
2. **Poll**: Check job status periodically
3. **Retrieve**: Get results when job completes

The SDK provides two approaches:

- **Automatic**: Use `*_and_wait()` methods (recommended for most cases)
- **Manual**: Use separate submit and polling methods for more control

### Response Models

All API responses are returned as typed dataclass objects:

- `ParseResponse`: Parse operation results
- `ExtractResponse`: Extraction results with confidence scores
- `ClassifyResponse`: Classification results
- `SplitResponse`: Document splitting results
